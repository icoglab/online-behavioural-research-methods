Introduction

This textbook is designed to teach you how to create sophisticated online behavioural experiments using modern web technologies, with a particular focus on jsPsych, a powerful JavaScript library specifically built for psychological research. Originally developed for my Research in Cognitive Psychology course at the University of Manitoba, this book provides a comprehensive introduction to programming online studies that can be used by anyone interested in conducting behavioural research on the web.

While the examples and labs throughout this book draw primarily from cognitive psychology research, reflecting my background as a cognitive psychologist, the principles and techniques you’ll learn are broadly applicable to any field conducting online behavioural studies. Whether you’re interested in social psychology, developmental psychology, neuroscience, or related disciplines, the skills you develop here will transfer directly to your research needs.

Book Structure and Learning Path

The book is structured to take you from complete beginner to competent online experiment programmer through a carefully scaffolded progression. You’ll start by learning the fundamental web technologies that power all online experiments: HTML for structure, CSS for styling, and JavaScript for interactivity. These foundational skills are essential because jsPsych, like all web-based experiment platforms, builds upon these core technologies.

Once you have a solid grasp of the fundamentals, you’ll dive into jsPsych itself, learning how this specialized library can dramatically simplify the process of creating psychological experiments. You’ll discover how to present stimuli, collect responses, measure precise timing data, and manage the flow of your experiments through jsPsych’s intuitive plugin system and timeline structure.

As you progress through the units, you’ll tackle increasingly sophisticated experimental designs. You’ll learn to incorporate multimedia elements like images, audio, and video into your studies. You’ll master techniques for creating dynamic, responsive experiments that adapt based on participant responses. You’ll explore methods for properly saving and organizing your data, implementing randomization strategies, and even adding gamification elements to enhance participant engagement.

Each unit combines theoretical instruction with hands-on laboratory exercises that reinforce your learning through practical application. These labs guide you through creating experiments that investigate classic phenomena in cognitive psychology, from memory and attention to decision-making and face perception. By working through these exercises, you’ll not only learn programming techniques but also gain insights into experimental design principles that are crucial for conducting rigorous behavioural research.

Prerequisites

This textbook is designed to be accessible to complete beginners with no prior programming experience. The only requirements are:

Technical Requirements:

  • A computer, tablet, or smartphone with internet access
  • A modern web browser (Chrome, Firefox, Safari, or Edge)
  • Ability to create accounts on websites (you’ll need a GitHub account)

No Software Installation Required: Everything we’ll do happens in your web browser through GitHub’s online interface. You won’t need to download or install any programming software, text editors, or development tools.

Academic Background: While the examples focus on cognitive psychology, no specific psychology background is required. The book explains psychological concepts as they arise, making it suitable for students from any discipline interested in online behavioral research.

Time Commitment: This book is designed around a 13-week course structure, with each chapter and associated lab tutorial intended to be completed within one week. Most students should expect to spend 3-5 hours per week on reading, exercises, and lab work, though this may vary depending on your familiarity with the concepts and your learning pace.

Learning Outcomes

By the end of this book, you’ll have the skills to design, implement, and deploy sophisticated online behavioural experiments. More importantly, you’ll understand the underlying principles that will allow you to adapt these techniques to your own research questions and continue learning as web technologies evolve. Whether you’re planning to conduct your own research, collaborate with other investigators, or simply want to understand how online studies work, this book will provide you with a solid foundation in the art and science of web-based behavioural research.

Getting Setup Online

For this book, we’re going to adopt the quickest and simplest method to get you started by using GitHub, a website that many people use to store and edit code. GitHub is an online platform that serves two main purposes for our course: it’s a place to store and manage your code files, and it can also host your experiments as live websites.

Think of GitHub as a sophisticated filing cabinet for your experiment files. When you create a repository (or “repo”), you’re creating a folder that can hold all the files for a particular project. GitHub keeps track of every change you make to these files, creating a complete history of your work. This means you can always go back to an earlier version if something breaks.

Step 1: Sign up for a free GitHub account

First, navigate to www.github.com and sign up for a free account. Keep in mind that the username you choose here will become part of your website address that you will use for the rest of the course. For example, if you choose “jsmith” as your username, your website will be https://jsmith.github.io/

Choose a professional username that you wouldn’t mind sharing with research collaborators or on your CV, as GitHub profiles often serve as portfolios for technical work.

Step 2: Fork the lab materials repository

The fork feature allows you to make your own copy of someone else’s repository. In this course, you’ll fork the lab materials repository, giving you your own personal copy of all the starter files and examples to work with.

You can find the lab materials here: https://github.com/icoglab/obrm-lab-materials. At the top of the screen, you will see a ‘Fork’ button. Click that and you will become the proud new owner of a copy of the lab starter materials.

Step 3: Turn your repo into a live website

GitHub offers a service called GitHub Pages that can turn your repository into a live website. This means the HTML, CSS, and JavaScript files in your repository become accessible to anyone on the internet through a web address. For example, if your GitHub username is “jsmith” and your repository is called “obrm-lab-materials”, your experiment would be available at https://jsmith.github.io/obrm-lab-materials.

This is incredibly convenient because you can edit your files directly on GitHub’s website using their built-in editor, and your changes appear on your live experiment website within minutes.

Now that you have a copy of the obrm-lab-materials repository saved in your account, you can activate it to turn into a live website. To do that, navigate to your new repository, which would be something like https://github.com/yourusername/obrm-lab-materials.

At the top of the screen is a Settings link. Click that and on the left you’ll find a Pages link. Under “Source,” select “Deploy from a branch” and choose “main” as your branch. Click “Save” and GitHub will provide you with the URL where your site will be published.

You will now be able to navigate to your site (e.g., https://username.github.io/obrm-lab-materials) and see the lab portfolio landing page.

Note: It may take a few minutes for your site to become available after enabling GitHub Pages.

Offline Setup

Alternatively, you can download the lab materials repo onto your own computer and edit the files using any text editor! When you open the html file in the browser (e.g., index.html), it will render your experiment as it would online.

Important Limitation: Data Storage

While GitHub Pages is excellent for hosting your experiment files, it has one important limitation: it cannot save participant data. GitHub Pages only serves static files, meaning it can display your experiment and collect responses temporarily, but it cannot permanently store that data on a server.

For the exercises in this course, this limitation won’t prevent you from learning and testing your experiments. You can still create fully functional experiments that collect responses, you just won’t be able to save that data permanently using GitHub Pages alone.

Alternative Hosting Options

University of Manitoba Server Access

University of Manitoba students have access to web hosting through the university’s servers, which can both host your experiment files and save participant data. Details for setting up and using this option are provided in the appendix of this textbook.

Other Hosting Solutions

Many other platforms can host web experiments and handle data storage, including commercial services like Netlify, Vercel, and traditional web hosting providers. Some are free for basic use, while others require payment. The choice of hosting platform often depends on your specific research needs, budget, and technical requirements.

Saving Data to OSF

Another solution to the data saving problem is to connect your GitHub-hosted website to the Open Science Framework repository using DataPipe. This allows you to keep hosting your experiments on GitHub and have the data files automatically save to an OSF repository. Instructions on how to do that can be found here: https://pipe.jspsych.org/.

A Note About This Textbook

This is the first edition of this textbook, developed specifically for hands-on learning in an evolving field. As with any first edition, you may encounter errors, unclear explanations, or technical issues as web technologies and platforms change.

If you encounter problems:

  • Technical issues with code examples: Check that you’ve followed all steps exactly as written, then try refreshing your browser or waiting a few minutes (GitHub Pages can take time to update)

  • Errors or unclear content: Please report these by visiting the textbook’s GitHub repository at https://github.com/icoglab/obrm-textbook and clicking on the “Issues” tab to log a new issue

  • Suggestions for improvement: We welcome feedback about content that could be explained more clearly or additional topics that would be helpful

Your feedback helps improve this resource for future students. Don’t hesitate to report even minor issues like typos or broken links. Contributing to the improvement of educational resources is part of learning to work collaboratively in the open-source community that powers much of modern web development.

Version Information: This textbook will be updated regularly based on student feedback and changes in web technologies. Check the repository for the most current version and any important updates or corrections.

Getting Started Mindset

Remember that every expert programmer started exactly where you are now. The files you’ll be editing are just text with special formatting rules. Your browser is remarkably forgiving, so if something doesn’t work perfectly the first time, you can always edit the text and try again.

Think of programming your experiment like writing instructions for a very literal assistant. You need to be specific and clear, but the basic building blocks are straightforward. By the end of this course, you’ll be amazed at what you can create with just text files and a web browser.
